# 论文补充实验方案

## 实验 1：执行正确率 (pass@k) — 最重要

### 目的
验证生成代码不仅文本匹配，而且**真的能在 VM 上跑出正确结果**。

### 方法
1. 先用 `whisper run` 执行 eval.jsonl 里的预期代码，确认预期代码本身能跑通
2. 再用 `whisper run` 执行模型生成的代码
3. 对比输出是否一致
4. 对每个任务生成 5 个样本（temperature=0.8），计算 pass@1, pass@5

### 产出
- pass@1 表格（15 个任务几个通过）
- pass@5 数据
- 失败案例的错误类型分析

---

## 实验 2：公平 Token 对比

### 目的
修正之前 Python 端输出教学代码的问题

### 方法
**不用生成的代码**，用**手写的参考代码**做对比：
1. 对 25 个 benchmark 任务，手写最简 Python 代码
2. 对同样的任务，手写最简 Whisper 代码（已有 benchmark.jsonl）
3. 用模型的 tokenizer 分别计数
4. 结果无法被质疑

### 产出
- 公平的 token 对比表
- Whisper vs Python 的 token ratio

---

## 实验 3：数据规模消融

### 目的
展示训练数据量对准确率的影响

### 方法
分别用 250、500、750、1149 条数据训练 4 个模型，对比：
- Exact Match %
- Syntax Valid %
- pass@1

### 产出
- 折线图：数据量 vs 准确率
- 外推预测：10,000 条数据能到多少准确率

---

## 实验 4：Stdlib 消融

### 目的
量化标准库的贡献

### 方法
准备两份数据：
- A：含 stdlib 导入模式的（1149 条，当前版本）
- B：去掉所有 stdlib 的（纯内联写法，约 1100 条）
分别训练，对比 token 消耗和准确率

### 产出
- Stdlib 对 token 节省的贡献（之前算了 36.6%，再验证）
- Stdlib 对准确率的影响
